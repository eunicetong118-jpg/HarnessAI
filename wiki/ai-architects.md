# AI Architects (Software 2.0)

The shift from writing specific algorithms to designing search spaces and templates.

## The Paradigm Shift
- **Software 1.0**: Humans write a finite sequence of machine-interpretable instructions (algorithms).
- **Software 2.0**: Humans specify a **template** (search space) and constraints. The machine finds the optimal algorithm using data.

## The Role of the AI Architect
- **Constraint Design**: Shrinking the search space using domain knowledge (locality, symmetry, compositionality).
- **Efficiency**: Specialized templates (like CNNs) require less data than generic ones (like GPT-3) because they encode human priors.
- **Symbolic Gap**: Bridging the gap between continuous differentiable programming (ANNs) and discrete symbolic reasoning (Lambda Calculus).

## Key Principles
- **Genomic Bottleneck**: Using compression/hierarchical structures to enable rapid learning from minimal data.
- **Legibility**: Designing systems where the machine's search process and the resulting algorithms are understandable and steerable by humans.
