| Boundary | Description |
|----------|-------------|
| Client → API | QR data sent to client; TOTP tokens received from client |
| API → Database | Encrypted secrets stored in DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | Database | mitigate | AES-256-GCM encryption for all TOTP secrets |
| T-05-02 | Tampering | Enrollment Flow | mitigate | Verify TOTP token before enabling `totpEnabled` flag |
| T-05-03 | Spoofing | QR Generation | mitigate | Generate QR DataURL on server; never expose raw secret to client |
</threat_model>
